Script Loading Strategies

All the html on the page is loading in the order in which it appear. If you are using javascript to manipulate on the page(or more accurately the DOM) your code won't work if the javascript is loaded and parsed before the html you are trying to do something to


There are a few different strategies to make sure your javascript only run after your html is parsed

  • In the external javascript above which you deleted, the script element is placed at the bottom of the body of the document and therefore only runs after the rest of the html is parsed
  • In the external javascript example above, the script element is placed in the head of the document before the html body is parsed. But because we are using type "module" the code is treated as a module and the browser wait for all html to be processed before executing javascript module. (you could also placed external script at the bottom of the body. but if there is a lot of html and the network is slow, it may be take time before the browser can even start fetching and loading the script, so placing external script at the head is better)
  • If you still want to use non module script in the document head, which could block the whole page from displaying and could cause error because it execute before the html is parse:
    • For external script, we should add the defer(or if you don't need the html to be ready they are synced attribute on the script element)
    • For internal script, you should wrap the code in a domContentLoadedEventlistener
